CodeTether language runtime

Small scripts for serious Rust systems.

TetherScript lets Rust projects accept plugins, policies, validators, and agent-written behaviors without turning every change into a rebuild or every extension into a supply-chain event.

fn validate(change) {
    if change.files.len() == 0 {
        return err("no files changed")
    }

    if change.diff.contains("unsafe") {
        return err("unsafe needs explicit review")
    }

    return ok("ship it")
}
01

No dependency gravity

The runtime is designed as a zero-third-party-dependency Rust crate, so embedding it does not pull a new ecosystem into the host.

02

Agent changes stay reviewable

Agents can edit behavior in script files while humans review a compact policy surface instead of generated Rust diffs.

03

Capabilities over ambient power

Hosts decide what scripts can touch. Filesystem, HTTP, process, and host hooks are explicit boundaries.

Where it fits

First-class inside CodeTether, embeddable anywhere Rust runs.

Inside CodeTether: repository-local automation, policy checks, tool guards, and plugin hooks.

Inside Rust products: customer-specific behavior without recompiling the host binary.

Inside agent workflows: deterministic, auditable scripts that can be generated, tested, and constrained.
